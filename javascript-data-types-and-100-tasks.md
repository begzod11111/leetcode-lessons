# JavaScript: типы данных, строки, массивы, объекты и 100 задач

Этот материал нужен как спокойный фундамент перед асинхронностью, Node.js,
алгоритмами и реальными проектами. Цель простая: научиться видеть, с какими
данными вы работаете, какие операции для них подходят и когда значение
изменяется, а когда создается новое.

Практика находится в файле
[`javascript-basics-100-tasks.js`](./javascript-basics-100-tasks.js). Там 100
задач без готовых решений. Решайте их по очереди: сначала простые значения,
потом числа, строки, массивы, объекты и смешанные задачи.

## 1. Какие типы данных есть в JavaScript

В JavaScript есть примитивные типы и ссылочные типы.

### Примитивные типы

Примитив - это простое значение, которое не является объектом.

Основные примитивы:

1. `number` - числа: `10`, `3.14`, `-5`, `NaN`, `Infinity`.
2. `bigint` - очень большие целые числа: `9007199254740993n`.
3. `string` - строки: `"hello"`, `'JavaScript'`, `` `Hi, ${name}` ``.
4. `boolean` - логика: `true` или `false`.
5. `undefined` - значение не задано.
6. `null` - намеренно пустое значение.
7. `symbol` - уникальный идентификатор.

Пример:

```js
const age = 25; // number
const name = "Ada"; // string
const isStudent = true; // boolean
const empty = null; // null
let result; // undefined
```

Проверить тип можно через `typeof`:

```js
typeof 10; // "number"
typeof "hello"; // "string"
typeof true; // "boolean"
typeof undefined; // "undefined"
typeof 10n; // "bigint"
typeof Symbol("id"); // "symbol"
```

Есть важная историческая особенность:

```js
typeof null; // "object"
```

Это не значит, что `null` обычный объект. Просто так работает старое поведение
языка.

## 2. Number: обычные числа

Тип `number` используется и для целых чисел, и для дробных:

```js
const apples = 5;
const price = 19.99;
```

Частые операции:

```js
10 + 3; // 13
10 - 3; // 7
10 * 3; // 30
10 / 3; // 3.333...
10 % 3; // 1
2 ** 3; // 8
```

Полезные функции и методы:

```js
Math.round(4.6); // 5
Math.floor(4.9); // 4
Math.ceil(4.1); // 5
Math.max(3, 9); // 9
Math.min(3, 9); // 3
Number.isNaN(NaN); // true
```

`NaN` означает "Not a Number". Это результат некорректной математической
операции:

```js
Number("abc"); // NaN
0 / 0; // NaN
```

## 3. String: строки

Строка - это текст. Строки можно создавать разными кавычками:

```js
const a = "hello";
const b = 'hello';
const c = `hello`;
```

Шаблонные строки удобны для вставки значений:

```js
const name = "Ada";
const text = `Hello, ${name}`;
```

Строки имеют длину и индексы:

```js
const word = "JavaScript";

word.length; // 10
word[0]; // "J"
word[word.length - 1]; // "t"
```

Частые методы:

```js
"hello".toUpperCase(); // "HELLO"
"HELLO".toLowerCase(); // "hello"
"  text  ".trim(); // "text"
"JavaScript".includes("Script"); // true
"hello".startsWith("he"); // true
"hello".endsWith("lo"); // true
"one two".split(" "); // ["one", "two"]
["one", "two"].join(" "); // "one two"
```

Строки являются примитивами, но JavaScript временно дает им методы через
внутреннюю обертку. Поэтому `"hello".toUpperCase()` работает, хотя строка не
обычный объект.

Важно: строки неизменяемые. Нельзя изменить один символ внутри существующей
строки. Можно только создать новую строку.

```js
const word = "cat";
const newWord = "b" + word.slice(1); // "bat"
```

## 4. Boolean: логика

`boolean` имеет только два значения:

```js
true;
false;
```

Обычно boolean получается из сравнений:

```js
10 > 5; // true
10 === "10"; // false
5 <= 3; // false
```

Основные логические операторы:

```js
true && false; // false, И
true || false; // true, ИЛИ
!true; // false, НЕ
```

В условиях JavaScript приводит значения к boolean:

```js
if ("hello") {
  console.log("Строка непустая");
}
```

Ложные значения:

```js
false;
0;
-0;
0n;
"";
null;
undefined;
NaN;
```

Все остальные значения обычно истинные.

## 5. undefined и null

`undefined` значит, что значение не задано:

```js
let name;
console.log(name); // undefined
```

`null` обычно ставят вручную, когда хотят сказать: "значения сейчас нет":

```js
const selectedUser = null;
```

Практическое правило:

- `undefined` - значение забыли, не передали или еще не присвоили;
- `null` - пустое значение выбрано специально.

## 6. BigInt и Symbol коротко

`bigint` нужен для очень больших целых чисел:

```js
const big = 9007199254740993n;
```

Нельзя свободно смешивать `number` и `bigint`:

```js
// 1n + 1; // TypeError
1n + 1n; // 2n
```

`symbol` создает уникальное значение:

```js
const id = Symbol("id");
```

На начальном уровне достаточно знать, что это тоже примитив. Чаще всего
новичок сначала много работает с `number`, `string`, `boolean`, `null`,
`undefined`, массивами и объектами.

## 7. Ссылочные типы: объекты, массивы, функции

Все, что не примитив, обычно является объектом или связано с объектами.

### Object

Объект хранит пары "ключ - значение":

```js
const user = {
  name: "Ada",
  age: 25,
  isAdmin: false,
};

user.name; // "Ada"
user["age"]; // 25
```

Можно добавлять и менять свойства:

```js
user.city = "London";
user.age = 26;
```

### Array: списки в JavaScript

В JavaScript список обычно называется массивом (`Array`):

```js
const numbers = [1, 2, 3];
const names = ["Ada", "Grace", "Linus"];
```

Индексы начинаются с нуля:

```js
names[0]; // "Ada"
names[1]; // "Grace"
names[names.length - 1]; // "Linus"
```

Частые методы:

```js
const items = [1, 2, 3];

items.push(4); // добавить в конец
items.pop(); // удалить с конца
items.unshift(0); // добавить в начало
items.shift(); // удалить с начала
items.includes(2); // true
items.indexOf(3); // 2
items.join(", "); // "1, 2, 3"
```

Методы для обработки:

```js
const numbers = [1, 2, 3, 4];

numbers.map((num) => num * 2); // [2, 4, 6, 8]
numbers.filter((num) => num % 2 === 0); // [2, 4]
numbers.reduce((sum, num) => sum + num, 0); // 10
```

### Function

Функция - это блок кода, который можно вызвать:

```js
function add(a, b) {
  return a + b;
}

add(2, 3); // 5
```

Стрелочная функция:

```js
const add = (a, b) => a + b;
```

Функции в JavaScript тоже являются значениями: их можно передавать в другие
функции, хранить в переменных и возвращать.

## 8. Значение и ссылка

Примитивы копируются как значения:

```js
let a = 10;
let b = a;
b = 20;

console.log(a); // 10
console.log(b); // 20
```

Объекты и массивы копируются как ссылки:

```js
const first = { count: 1 };
const second = first;

second.count = 2;

console.log(first.count); // 2
```

Если нужна копия массива:

```js
const original = [1, 2, 3];
const copy = [...original];
```

Если нужна копия объекта:

```js
const user = { name: "Ada", age: 25 };
const copy = { ...user };
```

На начальном уровне важно привыкнуть: массивы и объекты могут измениться через
другую переменную, если обе переменные смотрят на одно и то же место в памяти.

## 9. Преобразование типов

JavaScript иногда преобразует типы автоматически:

```js
"5" + 1; // "51"
"5" - 1; // 4
```

Для обучения лучше делать преобразование явно:

```js
Number("5"); // 5
String(5); // "5"
Boolean(1); // true
```

Для строгого сравнения используйте `===` и `!==`:

```js
5 === "5"; // false
5 == "5"; // true, но новичку лучше избегать
```

Практическое правило: в обычном коде почти всегда выбирайте строгое сравнение.

## 10. Как решать 100 задач

1. Откройте `javascript-basics-100-tasks.js`.
2. Решайте задачи сверху вниз.
3. В каждой задаче заменяйте `return undefined;` на свое решение.
4. Проверяйте себя через `console.log` или простые `if`-проверки.
5. После решения присылайте код на проверку: можно будет улучшить стиль,
   обработать крайние случаи и сделать рефакторинг.

Группы задач:

- 1-10: переменные, типы, базовые значения;
- 11-35: числа и простая математика;
- 36-55: строки;
- 56-80: массивы;
- 81-90: объекты;
- 91-100: смешанные задачи на массивы, строки и объекты.

Сначала не гонитесь за коротким кодом. Лучше написать понятное решение, а уже
после этого улучшать его.
